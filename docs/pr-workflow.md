# PR Workflow — Deterministic Agent Loop

## Overview

All code changes go through pull requests. No direct pushes to `main`.

The system implements the **Code Factory** pattern:
1. Risk gate classifies changes → determines required checks
2. Path-filtered CI runs only relevant tests
3. Automated code review for high/medium risk PRs
4. Discord approval with Components v2 interactive buttons
5. Merge only after all gates pass

## Risk Tiers

Defined in `risk-policy.json` (the single source of truth):

| Tier | Files | Required Checks |
|------|-------|----------------|
| 🔴 HIGH | Dashboard, CI workflows, deploy scripts | risk-gate + test + lint + review |
| 🟡 MEDIUM | Pipelines, models, scrapers | risk-gate + test + lint |
| 🟢 LOW | Docs, markdown | risk-gate + lint |

## CI Workflows (Path-Filtered)

| Workflow | Triggers On | Runs |
|----------|------------|------|
| `ci-projects.yml` | nhl-betting/, polymarket/, real-estate/ | Lint + test affected projects |
| `ci-dashboard.yml` | admin-dashboard/ | Backend lint+test, frontend build |
| `ci.yml` | scripts/, .github/, Makefile, pyproject.toml | Full suite |
| `risk-policy-gate.yml` | All PRs | Risk classification + summary comment |
| `pr-review.yml` | All PRs | Automated code review request |

## Schema Protection

Database schema changes (CREATE/ALTER/DROP TABLE, migration files) are flagged by the risk gate.
If a project changes DB schema, the admin dashboard must be updated in a companion PR to verify compatibility.

The dashboard auto-discovers tables from all configured databases — but column changes or table drops
can break existing queries and UI.

## Discord Integration

PRs post interactive cards to Discord with:
- Risk tier badge
- Approve / Request Changes buttons
- Direct link to PR

Generated by `scripts/pr-discord-notify`.

## Agent Instructions

All coding agents (Claude Code, Codex CLI) MUST:
1. Work on feature branches (`feat/`, `fix/`, etc.)
2. Never push directly to `main`
3. Open PRs via `gh pr create`
4. Wait for CI and review before merging

## Scripts

- `scripts/risk-classifier` — classify files by risk tier
- `scripts/pr-discord-notify` — generate Discord PR review cards
- `scripts/committer` — safe commit helper
- `scripts/deploy-dashboard` — blue-green deployment (dashboard only)
