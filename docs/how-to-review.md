# How Code Reviews Work

## The Simple Version

1. Someone pushes code to a branch
2. They open a Pull Request (PR) on GitHub
3. NHC's bot reads the code and posts a review comment on the PR
4. If it finds problems → it tells you what to fix
5. If it's clean → it auto-approves
6. A human (Blake, Ian, or Connor) clicks "Merge" — the bot can't merge

## How to Run a Review

### Step 1: Load the API keys
```bash
cd ~/nhc-capital
export $(grep -v '^#' admin-dashboard/.env | xargs)
```
This loads the NVIDIA + OpenRouter API keys from the `.env` file. You only need to do this once per terminal session.

### Step 2: Find the PR number
Go to https://github.com/nhccapitalinc-gif/nhc-capital/pulls

The number is the `#XX` — like `#20`, `#21`, etc.

Or from the terminal:
```bash
cd ~/nhc-capital
gh pr list
```

### Step 3: Run the review
```bash
scripts/mr-review 20
```
Replace `20` with whatever PR number you want to review.

That's it. The bot will:
- Read the code changes
- Classify risk (high/medium/low)
- Run regex checks (secrets, SQL injection, etc.)
- Send the diff to an AI model for deep review
- Post the review as a comment on the PR in GitHub

### Options
```bash
scripts/mr-review 20                  # Basic review
scripts/mr-review 20 --auto-resolve   # Also clean up old bot threads
scripts/mr-review 20 --remediate      # Show findings for a coding agent to fix
```

## What Happens Behind the Scenes

```
You run: scripts/mr-review 20
         │
         ├─ Fetches the PR diff from GitHub
         ├─ Classifies changed files by risk tier
         ├─ Runs regex pattern checks (fast, catches obvious stuff)
         │
         ├─ Sends diff to AI model for deep review:
         │   Try Kimi K2.5 (20s) ──failed──►
         │   Try GLM5 (20s) ──failed──►
         │   Try OpenRouter free (Qwen3, Mistral, etc.) ──failed──►
         │   Try DeepSeek V3.2 (20s, last resort)
         │
         ├─ Posts review comment on the PR (tagged with commit SHA)
         ├─ If clean: auto-approves the PR
         └─ If critical issues: blocks approval
```

## FAQ

**Q: Do I have to run it manually every time?**
A: Right now, yes. Automatic triggering on PR push is on the TODO list.

**Q: What if all the AI models are down?**
A: The regex checks still run (they catch secrets, SQL injection, DDL). The review just won't have the deep AI analysis.

**Q: Can the bot merge code?**
A: No. NHC has limited permissions — it can review and approve, but only Owners (Blake, Ian, Connor) can click Merge.

**Q: Where are the API keys?**
A: `~/nhc-capital/admin-dashboard/.env` (gitignored, never committed). GITHUB_TOKEN is set via `gh auth login`.

**Q: What's a SHA?**
A: A commit ID — like `5b30764`. Every time you push new code, the SHA changes. The bot tags its review with the SHA so old reviews don't count for new code.
